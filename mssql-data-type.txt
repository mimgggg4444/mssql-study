숫자 데이터 유형:

INT: 정수 값을 저장하며, 4바이트 크기입니다.
BIGINT: 큰 정수 값을 저장하며, 8바이트 크기입니다.
SMALLINT: 작은 정수 값을 저장하며, 2바이트 크기입니다.
TINYINT: 아주 작은 정수 값을 저장하며, 1바이트 크기입니다.
DECIMAL(p, s): 고정 소수점 숫자를 저장하며, 전체 자릿수(p)와 소수점 이하 자릿수(s)를 지정합니다.
NUMERIC(p, s): DECIMAL과 동일한 기능을 제공합니다.
FLOAT: 부동 소수점 숫자를 저장하며, 8바이트 크기입니다.
REAL: 부동 소수점 숫자를 저장하며, 4바이트 크기입니다.


문자열 데이터 유형:

CHAR(n): 고정 길이 문자열을 저장하며, 최대 8,000자까지 저장 가능합니다.
VARCHAR(n): 가변 길이 문자열을 저장하며, 최대 8,000자까지 저장 가능합니다.
NCHAR(n): 유니코드 고정 길이 문자열을 저장하며, 최대 4,000자까지 저장 가능합니다.
NVARCHAR(n): 유니코드 가변 길이 문자열을 저장하며, 최대 4,000자까지 저장 가능합니다.
TEXT: 대용량 텍스트 데이터를 저장하며, 최대 2GB까지 저장 가능합니다.
NTEXT: 유니코드 대용량 텍스트 데이터를 저장하며, 최대 2GB까지 저장 가능합니다.


날짜와 시간 데이터 유형:

DATE: 날짜 값을 저장하며, YYYY-MM-DD 형식으로 표현합니다.
TIME: 시간 값을 저장하며, HH:MM:SS 형식으로 표현합니다.
DATETIME: 날짜와 시간 값을 저장하며, YYYY-MM-DD HH:MM:SS 형식으로 표현합니다.
SMALLDATETIME: 날짜와 시간 값을 저장하며, YYYY-MM-DD HH:MM:SS 형식으로 표현하지만, 정확도가 1분입니다.


이진 데이터 유형:

BINARY(n): 고정 길이 이진 데이터를 저장하며, 최대 8,000바이트까지 저장 가능합니다.
VARBINARY(n): 가변 길이 이진 데이터를 저장하며, 최대 8,000바이트까지 저장 가능합니다.
IMAGE: 대용량 이진 데이터를 저장하며, 최대 2GB까지 저장 가능합니다.


기타 데이터 유형:

BIT: 0 또는 1의 값을 저장하는 불리언 데이터 유형입니다.
MONEY: 화폐 값을 저장하며, 소수점 이하 4자리까지 표현합니다.
UNIQUEIDENTIFIER: GUID(Globally Unique Identifier)를 저장하는 데 사용됩니다.
XML: XML 데이터를 저장하는 데 사용되며, 최대 2GB까지 저장 가능합니다.
